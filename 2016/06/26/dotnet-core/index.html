dotnet core

发表于 2016-06-26   |   分类于 .NET   |  

2016年初的时候ghost在非Windows平台上的.NET开发一文中介绍了基于DNX的.NET Core跨平台开发。而现在，微软正式发布了.NET Core，而不是依赖DNX，因此我觉得有必要更新一下文章了。

首先，.NET Core相关信息的网站有如下几个，快速浏览一下对您的理解会有很好的帮助：

  • .NET Core: https://www.microsoft.com/net/core
  • .NET Core Github: https://dotnet.github.io/

本文介绍的重点是跨平台.NET Core的开发，因此以非Windows平台为主介绍。 在windows平台下使用.NET Core，对于广大.NET开发者来说完全不是问题。

本文所有的操作都以MacOS X为例，Linux平台下虽有略微差异，但过程大致相同。如您有任何在Linux平台操作.NET Core的困难，欢迎随时联系我。

第一件事情不是如何安装.NET Core，而是如何卸载老的DNX和DNVM。DNX的存在并不影响新的.NET Core的使用，但是为了不占硬盘您可以考虑删除。

删除DNX

DNVM可能在电脑中安装多个DNX版本，可以通过dnvm list命来来进行查看

可以看到，ghost的电脑中安装有两个不同版本的DNX。要删除它们可以用DNVM uninstall命令来完成，但是那个命令实在是太难用了。事实上，所有的DNX版本都安装在~/.dnx目录下，只需把这个目录删除就达到目的了。

虽然我们还能从Terminal中看到DNVM命令能执行，但那只是当前会话中的一个缓存，重启Terminal就完成了DNVM和DNX的删除了。

##.NET Core的安装
链接https://www.microsoft.com/net/core 对不同平台下.NET Core的安装有了详细的说明，ghost就不需要再重复了. :-)

安装完.NET Core后，所有的功能都是通过dotnet这个命令来完成的。快来体验一下吧。

dotnet的体验

在demo之前，先简要介绍一下dotnet的常用命令：

  • dotnet --help dotnet的命令帮助
  • dotnet new 新建一个.NET Core的工程
  • dotnet restore .NET Core的工程是通过nuget来管理包的依赖关系的，通过这个命令来下载所有的依赖包。
  • dotnet run 运行当前.NET Core工程

想查看更多命令？快去查看dotnet --help吧。现在让我们来看Demo吧。

恩，就这么简单，基于.NET Core的Hello World程序就在MacOS X上完成了。

从玩具到项目工程

写一个Hello World的程序那只是一个玩具，不代表有能力进行项目工程的开发。微软当然理解这一点，于是派出了设计模式之父Eric Gamma开发出了神一样的产品Visual Studio Code。

有了完全跨平台产品Visual Studio Code的支持，.NET Core在跨平台下的开发具备了重要的功能：

  • 调试
  • 智能感知
  • 代码分析

有了这些功能，开发一个项目还难吗？

顺便提一下，Visual Studio Code的功能完善真是日新月异啊，不久前在Mac平台上安装后还要手动运行脚本才能把Code添加到Path里，而现在可以直接从Code中设置了，真是体贴啊！
